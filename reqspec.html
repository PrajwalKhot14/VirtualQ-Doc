Functional Requirements

  • The user would want to view the number of patients ahead of the user
  • The user would want to see an approximate wait time based on the number of patients ahead of him/her
  • The user should be able to leave the queue anytime
  • The user should be able to down the queue anytime
  • If a user leaves the queue, other users should be given the option to be bumped up the queue
  • The user should be able to able to search for things to do around the hospital during the waiting time available
  • The user should be able to view the results of the information search
  • The user should be able to view the time continuously on the screen no matter what stage of the search they are on
  • The user should be notified when the wait time is almost coming to an end

Usability Requirements

  • An internet connection must be active on the user's mobile phone
  • Voice commands used to perform actions must be in simple English
  • Easy to learn
  • To ensure safety of the application, if the user chooses to leave the waiting room, the user should reconfirm the action
  • The user should know how to enter code provided by the admin of the waiting room to enter a specific waiting room
  • The user should be provided with help tutorials
